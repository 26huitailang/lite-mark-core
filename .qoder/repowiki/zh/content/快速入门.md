## 简介

LiteMark 是一个轻量级的照片参数水印工具，专为摄影爱好者设计。它能够自动提取照片的EXIF数据（拍摄参数），并将其添加到图片底部的水印框中。无论你是专业摄影师还是摄影爱好者，LiteMark 都能帮助你快速为照片添加专业的水印。

### 主要特性
- ✨ 自动提取照片拍摄参数（ISO、光圈、快门速度等）
- 🎨 支持多种预设模板（经典、现代、极简）
- 📸 批量处理功能
- 🔒 隐私保护：所有处理都在本地完成
- 🎯 简洁的命令行界面

## 安装准备

### 系统要求
- 支持的操作系统：Windows、macOS、Linux
- 至少 50MB 可用磁盘空间
- 支持的图片格式：JPEG、PNG

### 下载安装
```bash
# 方法1：从源码编译（需要 Rust 环境）
git clone https://github.com/26huitailang/lite-mark-core.git
cd lite-mark-core
cargo build --release

# 方法2：下载预编译二进制文件
# 访问 https://github.com/26huitailang/lite-mark-core/releases 下载对应平台的版本
```

### 验证安装
```bash
# 检查 LiteMark 是否安装成功
litemark --version
```

## 第一个水印操作

让我们从最简单的单张图片处理开始。这是你第一次使用 LiteMark 的最佳起点。

### 基础命令格式
```bash
litemark add -i INPUT_IMAGE -t TEMPLATE -o OUTPUT_IMAGE
```

### 参数详解
- `-i, --input`：输入图片文件路径（必需）
- `-t, --template`：使用的模板名称或路径（可选，默认为 "classic"）
- `-o, --output`：输出图片文件路径（必需）

### 实际操作示例

#### 步骤1：准备测试图片
首先，在你的电脑上创建一个测试文件夹，放入一张你喜欢的照片，比如 `photo.jpg`。

#### 步骤2：执行基础水印命令
```bash
# 使用经典模板为图片添加水印
litemark add -i photo.jpg -t classic -o photo_watermarked.jpg
```

#### 步骤3：查看结果
执行完成后，你会在当前目录看到生成的 `photo_watermarked.jpg` 文件。打开它，你会看到：
- 底部有一个半透明的黑色矩形水印框
- 显示了摄影师姓名和基本拍摄参数（ISO、光圈、快门速度）

**重要提示**：如果你的图片没有显示预期的水印，请检查以下几点：
- 确认图片文件存在且格式正确
- 确认输出路径有写入权限
- 检查是否有足够的磁盘空间

## 探索内置模板

LiteMark 提供了三种内置模板，每种都有独特的风格和用途。

### 查看可用模板
```bash
# 列出所有可用的模板
litemark templates
```

### 经典模板（ClassicParam）
```bash
# 使用经典模板
litemark add -i photo.jpg -t classic -o photo_classic.jpg
```

**特点**：
- 位置：左下角
- 内容：摄影师姓名 + 基本拍摄参数
- 背景：半透明黑色矩形

### 现代模板（Modern）
```bash
# 使用现代模板
litemark add -i photo.jpg -t modern -o photo_modern.jpg
```

**特点**：
- 位置：右上角
- 内容：相机型号 + 镜头 + 拍摄参数
- 背景：半透明黑色矩形

### 极简模板（Minimal）
```bash
# 使用极简模板
litemark add -i photo.jpg -t minimal -o photo_minimal.jpg
```

**特点**：
- 位置：右下角
- 内容：仅摄影师姓名
- 背景：无

### 查看模板详情
```bash
# 查看特定模板的详细配置
litemark show-template classic
litemark show-template modern
litemark show-template minimal
```

## 批量处理

当需要处理多个图片时，批量处理功能可以大大提高效率。

### 基础批量命令
```bash
# 批量处理整个目录
litemark batch -i INPUT_DIRECTORY -t TEMPLATE -o OUTPUT_DIRECTORY
```

### 参数说明
- `-i, --input-dir`：输入图片目录路径（必需）
- `-t, --template`：使用的模板名称（可选，默认为 "classic"）
- `-o, --output-dir`：输出目录路径（必需）

### 实际操作示例

#### 步骤1：准备图片目录
创建一个包含多张照片的文件夹，例如 `photos/`，里面放入多张 JPG 或 PNG 格式的图片。

#### 步骤2：执行批量处理
```bash
# 使用经典模板批量处理
litemark batch -i photos/ -t classic -o watermarked_photos/
```

#### 步骤3：查看处理结果
命令执行完成后：
- 输出目录 `watermarked_photos/` 将包含所有处理后的图片
- 每张图片都会被命名为原文件名后加上 `_watermarked` 后缀

### 批量处理的优势
- **效率提升**：一次性处理数百张图片只需几秒钟
- **一致性保证**：所有图片使用相同的模板和设置
- **节省时间**：避免重复手动操作

## 自定义作者名称

除了使用默认的 EXIF 数据中的作者信息，你还可以自定义水印中的作者名称。

### 基础自定义命令
```bash
# 添加自定义作者名称
litemark add -i photo.jpg -t classic -o photo_custom.jpg --author "John Doe"
```

### 参数详解
- `--author`：自定义作者名称（覆盖 EXIF 中的作者信息）

### 实际操作示例

#### 示例1：使用个人名字
```bash
litemark add -i family_photo.jpg -t classic -o family_watermarked.jpg --author "张三"
```

#### 示例2：使用工作室名称
```bash
litemark add -i portrait.jpg -t modern -o portrait_watermarked.jpg --author "光影工作室"
```

#### 示例3：批量处理时自定义作者
```bash
# 批量处理并设置统一的作者名称
litemark batch -i photos/ -t classic -o output/ --author "专业摄影师"
```

### 作者名称的应用场景
- **品牌建设**：使用工作室或公司的名称
- **个人标识**：展示个人的摄影品牌
- **项目分类**：为特定项目或活动添加标识
- **版权保护**：明确作品的所有权归属

## 故障排除

在使用 LiteMark 过程中，可能会遇到一些常见问题。以下是详细的解决方案。

### 常见问题及解决方法

#### 1. 模板未找到错误
**错误信息**：`Template 'xxx' not found`

**可能原因**：
- 模板名称拼写错误
- 使用了不存在的模板名称

**解决方法**：
```bash
# 查看可用模板列表
litemark templates

# 使用正确的模板名称
litemark add -i photo.jpg -t classic -o output.jpg
```

#### 2. 图片无法加载
**错误信息**：`Failed to load image`

**可能原因**：
- 文件路径不正确
- 图片格式不受支持
- 文件损坏或已删除

**解决方法**：
```bash
# 检查文件是否存在
ls -la photo.jpg

# 确认图片格式支持
file photo.jpg

# 使用绝对路径
litemark add -i /full/path/to/photo.jpg -t classic -o output.jpg
```

#### 3. 输出文件无法保存
**错误信息**：`Permission denied` 或 `Cannot create output directory`

**可能原因**：
- 输出目录不存在
- 当前用户没有写入权限
- 磁盘空间不足

**解决方法**：
```bash
# 创建输出目录
mkdir -p output_directory

# 检查磁盘空间
df -h

# 使用相对路径
litemark add -i photo.jpg -t classic -o ./output.jpg
```

#### 4. 批量处理失败
**错误信息**：部分图片处理失败

**解决方法**：
```bash
# 检查输入目录中的图片格式
find photos/ -type f ! -name "*.jpg" ! -name "*.png"

# 分批处理大目录
litemark batch -i photos_part1/ -t classic -o output1/
litemark batch -i photos_part2/ -t classic -o output2/
```

### 获取帮助信息

#### 查看完整帮助
```bash
# 查看主命令帮助
litemark --help

# 查看特定子命令帮助
litemark add --help
litemark batch --help
litemark templates --help
litemark show-template --help
```

#### 调试模式
```bash
# 启用详细输出（如果支持）
litemark add -i photo.jpg -t classic -o output.jpg --verbose
```

### 性能优化建议

#### 大图片处理
- 对于超大图片（分辨率超过 4K），处理时间会相应增加
- 建议先缩小图片尺寸再添加水印

#### 批量处理优化
- 将图片按大小分组处理
- 在 SSD 上运行以获得更好的性能
- 避免同时运行其他占用大量磁盘 I/O 的程序
