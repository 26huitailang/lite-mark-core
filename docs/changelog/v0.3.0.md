# v0.3.0 - 深度重构：从 MVP 到发布标准

**发布日期**: 2026-04-20

## 概述

本次版本完成 LiteMark 从 MVP 到发布标准的深度改造，涵盖基础重构、布局引擎重构、模板系统升级和工程化完善四个阶段。所有 121 个测试通过，WASM 编译正常。

---

## Phase 1: 基础重构

### 结构化错误类型

- **新增** `litemark-core/src/error.rs`，定义 `CoreError` / `ImageError` / `ExifError` / `FontError` / `TemplateError` / `RenderError`
- 使用 `thiserror` 自动实现 `Error` trait
- 调用方可精确匹配错误类型（`CoreError::Font` vs `CoreError::Render`）
- 替换全项目 `Box<dyn std::error::Error>` → `CoreError`

### 字体引擎迁移: rusttype → ab_glyph

- 移除已停止维护的 `rusttype`，替换为官方继任者 `ab_glyph` (v0.2.32)
- 解决 `Box::leak` 内存泄漏问题
- 提升渲染质量（更好的曲线光栅化）
- 保留自定义字体注入能力

### 多字重字体系统

- `WatermarkRenderer` 从单字体改为 `FontSet { regular, bold }`
- `render_text_simple` 接收 `FontWeight` 参数并选择对应字体
- 粗体未加载时自动回退到常规体（不 panic）
- `weight` 字段正式生效（Normal / Bold / Light）

### 无文件系统依赖

- `create_builtin_templates()` 从文件读取改为 `include_str!` 编译时嵌入
- Core 真正实现"纯内存 API"的设计承诺

## Phase 2: 布局引擎重构

### 渲染模式系统（RenderMode）

新增 `RenderMode` 枚举，模板通过 `render_mode` 字段控制水印风格：

| 模式 | 效果 | 适用场景 |
|------|------|----------|
| `BottomFrame` | 底部纯白框 | 传统风格 |
| `GradientFrame` | 底部 25%→100% 白渐变 | 现代过渡，不割裂照片 |
| `Minimal` | 底部 1px 细线，无背景 | 极简风格 |
| `Overlay` | 照片右下角半透明暗底 | 不扩展画布，签名风格 |

### 排版质量提升

- **右对齐**: 右栏 EXIF 参数通过 `text_width()` 计算后真正右对齐
- **Logo 双线性缩放**: 替换最近邻插值，消除边缘锯齿
- **分隔线变淡**: `#C8C8C8` → `#E8E8E8`

### Alpha 颜色支持

- `parse_color` 支持 `#RRGGBBAA` 格式（最后两位为透明度）
- 背景、文字、分隔线均可设透明度
- 向后兼容（6 位 hex 默认 alpha=FF）

## Phase 3: 模板系统与视觉设计升级

### 四个内置模板重设计

| 模板 | 渲染模式 | 风格 |
|------|----------|------|
| **Classic** | BottomFrame | 纯白底框，三层信息层级 |
| **Compact** | Minimal | 底部细线，双行极简 |
| **Professional** | GradientFrame | 渐变过渡，四行信息 |
| **Overlay** *(新增)* | Overlay | 右下角半透明圆角黑底白字 |

### 视觉层级配色

- Level 1 (主信息): `#1A1A1A` bold
- Level 2 (次信息): `#4A4A4A` normal
- Level 3 (参数): `#6A6A6A` normal
- 分隔线: `#E8E8E8`

## Phase 4: 工程化与可维护性

### 视觉回归测试

- 新增 `tests/src/integration/visual_regression_tests.rs`
- 4 个内置模板在 1920×1080 下生成参考图（`tests/fixtures/expected/`）
- 像素级 diff，单通道容差 2，整体容差 0.5%
- 更新参考图：`UPDATE_REFS=1 cargo test --test integration -- visual`

### 性能基准测试

- 新增 `tests/benches/render_bench.rs`（Criterion）
- Release 模式结果：
  - 800×600: ~0.94 ms
  - 1920×1080: ~3.97 ms
  - 4000×3000: ~23.4 ms
- 目标 `< 200ms` 大幅超越

- `templates/TEMPLATE_GUIDE.md`: 添加 render_mode、alpha 颜色、4 模板对比
- `docs/DESIGN_SYSTEM.md` *(新建)*: 间距/颜色/字体规范、4 种渲染模式设计图

### 编译警告清理

- 自动修复冗余类型转换、可折叠 if、未使用导入
- clippy 警告从数十条降至 3 条设计层面警告（`too_many_arguments`）
