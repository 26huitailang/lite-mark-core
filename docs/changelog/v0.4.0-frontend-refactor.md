# v0.4.0 — 前端重构：Vue3 + Vite + TypeScript

**发布日期**: 2026-05-10

## 概述

将原有的单文件 `litemark-wasm/demo.html`（650 行内联 CSS/JS）重构为基于 Vue3 + Vite + TypeScript + Pinia 的现代化前端应用。重构后代码结构清晰、类型安全、可维护性大幅提升，并新增了前后对比、批量 ZIP 下载、可视化模板编辑等功能。

## 新增功能

### 前后图片对比 (ImageCompare.vue)

- 点击结果卡片上的 🔍 按钮打开全屏对比模式
- 水平滑块拖动查看处理前后的差异
- 支持鼠标拖拽和触摸滑动

### 批量 ZIP 下载

- 处理 2 张以上图片时显示"打包下载"按钮
- 使用 JSZip 将所有结果打包为 `litemark_batch_<timestamp>.zip`

### 可视化模板编辑 (VisualMode.vue)

- 表单化配置模板参数（滑块、下拉框、颜色选择器）
- 实时预览 JSON 输出
- 动态添加/删除/编辑内容项（items）

### JSON 编辑器 (JsonMode.vue)

- 集成 CodeMirror 6，支持语法高亮和暗色主题
- 实时验证 JSON 有效性
- 错误时保留上次有效配置作为 fallback

### localStorage 持久化

- 自动记忆：作者名称、上次使用的模板、编辑模式
- 作者历史记录（最近 10 个）

### 全局 Toast 通知

- 成功 / 错误 / 警告 / 信息 四种类型
- 自动 5 秒消失，最多同时显示 3 条

## 架构改进

### Pinia 状态管理

| Store | 职责 |
|-------|------|
| `files` | 文件选择、HEIC 过滤与转换、Logo 管理 |
| `template` | 模板配置、三模式同步、JSON 验证 |
| `processing` | WASM 处理状态、进度、结果集 |
| `settings` | localStorage 读写、作者历史 |

### WASM 集成

- `useWasm.ts` Composable 封装 `init()` 和 `process_batch()`
- `wasm-module.d.ts` 声明文件解决 wasm-pack 生成包不在仓库中的类型问题
- Vite alias `@wasm` 指向 `../pkg/`

### 类型安全

- 全项目 TypeScript，无 `any`
- `TemplateConfig` / `TemplateItem` / `ProcessResult` 等核心接口强类型
- `WasmError` 带阶段分类（init/decode/exif/template/render/encode）

## 遗留事项与已知限制

### 1. 测试未实现

Vitest 已配置但测试用例未编写。需要补充：
- Composables 测试（`useHeic`, `useLocalStorage`）
- Store 单元测试（`templateStore` 模式切换、`filesStore` HEIC 过滤）
- 关键组件测试（`UploadArea` 拖拽事件、`TemplateEditor` 标签切换）

### 2. TypeScript 类型检查跳过

`vue-tsc` 与 Node.js 25 不兼容，构建脚本已改为直接用 `vite build`。类型检查需通过 `npm run typecheck` 手动执行（修复 vue-tsc 兼容性后可用）。

### 3. WASM 同步处理阻塞 UI

`process_batch` 在 JS 主线程同步执行，处理大量高分辨率图片时界面会冻结。后续可考虑：
- 使用 Web Worker 包装 WASM 调用
- 或单张串行处理 + `requestAnimationFrame` 让步

### 4. JSON 编辑器无 Schema 验证

CodeMirror 6 仅提供语法高亮，无 JSON Schema 验证。用户可能输入符合 JSON 语法但不符合模板结构的配置。

### 5. 图片压缩选项缺失

ZIP 下载使用原始输出字节，未提供压缩率调节。对于批量处理大图场景，文件体积可能较大。

### 6. 移动端体验待优化

部分组件在移动端（如可视化编辑器的双栏布局、对比模式的触摸交互）体验可以进一步打磨。

### 7. WASM 错误分类基于字符串匹配

`parseWasmError` 通过错误消息关键词推断错误阶段，可能误分类。理想情况下 WASM crate 应返回结构化错误代码。
