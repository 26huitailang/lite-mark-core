# 需求分析：文字渲染分辨率与图片分辨率不匹配

## 1. 现状分析

当前文字渲染的核心逻辑位于 `litemark-core/src/renderer.rs`，其缩放机制如下：

| 元素 | 计算方式 | 硬编码限制 |
|------|---------|-----------|
| 边框高度 `frame_height` | `short_edge × frame_height_ratio` | `clamp(80, 800)` px |
| 字体大小 `font_size` | `frame_height × font_size_ratio` | `clamp(10, 100)` px |
| 内边距 `padding` | `frame_height × padding_ratio` | `clamp(5, 50)` px |

**流程**：先按图片**短边**计算边框高度，再按边框高度比例计算字体大小。所有最终值都被强制限制在固定的绝对像素范围内。

---

## 2. 问题根因

### 2.1 核心瓶颈：100px 字体硬上限

这是导致"不匹配"的**最直接原因**。

以 `professional` 模板处理一张 **6000×4600** 图片为例：

- 图片短边 = 4600px
- 边框高度 = `4600 × 0.15` = **690px**
- 主标题字体设计比例 = 0.26 → 理论大小 = `690 × 0.26` = **179px**
- 实际渲染大小 = `min(179, 100)` = **100px**

**结果**：文字仅占边框高度的 **14.5%**，远低于模板设计的 **26%**。在 6000px 宽的高分辨率图片上，100px 的文字视觉上显得非常小，与宽大的边框和整体图片分辨率严重失调。

同理，secondary font（0.16 比例 → 110px）也被压到 100px，进一步加剧了信息层级模糊。

### 2.2 800px 边框硬上限的级联影响

对于更高分辨率的图片（如全景图、8K），边框高度会在 800px 处被截断。此时即使 `frame_height_ratio=0.15`，边框也无法继续增长，但文字仍然受 100px 限制，相对失调问题会被进一步放大。

### 2.3 缺乏"分辨率基准"缩放机制

当前系统隐含假设所有输入图片都在一个"标准分辨率"范围（约 800px ~ 4000px 短边）内。当处理现代相机常见的 6000px 级别图片时，没有**基于图片总分辨率的动态缩放因子**来调整绝对限制，导致所有高分辨率输出都共用相同的像素天花板。

---

## 3. 影响范围

- **高分辨率照片**：6000×4000 及以上的相机原片，professional/classic 模板文字明显过小。
- **模板设计意图失效**：模板 JSON 中定义的 `font_size_ratio`（如 0.26）在高分辨率图上无法生效，设计者无法通过模板参数控制大图的视觉表现。
- **用户体验一致性差**：同一张图缩放到不同分辨率后，文字相对比例不一致。例如 1920px 短边的图文字比例正常，但 4000px 短边的图文字比例突然"塌陷"。
- **相关代码文件**：
  - `litemark-core/src/renderer.rs`：`render_frame_content()` 中的 `clamp` 逻辑
  - `litemark-core/src/layout.rs`：`Template` 结构定义（如需扩展配置）

---

## 4. 改进方案

### 方案 A：移除硬编码上限（短期，推荐）

**做法**：将 `font_size.max(10.0).min(100.0)` 改为仅保留下限，或大幅提高上限。

```rust
// 当前
let font_size = font_size.max(10.0).min(100.0);

// 建议：移除上限，或改为与图片短边关联的动态上限
let font_size = font_size.max(10.0);
```

**优点**：最简单，直接恢复模板设计比例。
**缺点**：极低分辨率图已有 10px 下限保护，但需验证超大分辨率图（如 10000px）是否会导致字体过大。

### 方案 B：引入基准分辨率缩放因子（中期）

**做法**：以 **1920px 短边**作为基准分辨率（1.0x），计算缩放因子：

```rust
let resolution_scale = (short_edge / 1920.0).max(1.0); // 只放大，不缩小
let max_font_size = 100.0 * resolution_scale;
let max_frame_height = 800.0 * resolution_scale;
```

**优点**：保证在不同分辨率设备上查看时，视觉大小保持一致。
**缺点**：需要确定合理的基准值，且可能影响现有低分辨率图的行为。

### 方案 C：模板级分辨率配置（长期）

在 `Template` 结构中添加可选字段：

```json
{
  "resolution_reference": 1920,
  "max_font_size": 0,
  "enable_resolution_scaling": true
}
```

允许每个模板独立控制是否启用分辨率感知渲染。

---

## 5. 建议的下一步

1. **立即执行方案 A**：移除 `renderer.rs` 中的 `min(100.0)` 字体上限，验证 professional/compact 模板在 6000px 图片上的效果。
2. **同步调整**：考虑将 `bottom_frame_height` 的 `clamp(80, 800)` 和 `padding` 的 `clamp(5, 50)` 也改为基于分辨率的比例限制，避免"大边框小文字"的失调。
3. **生成对比测试**：用同一张高分辨率原片分别生成"修改前/修改后"的输出，直观对比文字与边框的比例是否协调。
