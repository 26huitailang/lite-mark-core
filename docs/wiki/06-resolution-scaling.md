# 分辨率、缩放与响应式设计

> LiteMark 的模板系统天然支持"任意分辨率"，但这背后有一些重要的设计决策和权衡。理解分辨率缩放，能帮助你设计更好的模板和排查显示问题。

---

## 1. 分辨率基础

### 1.1 什么是分辨率

**分辨率** = 图像的宽度 × 高度，单位为**像素（pixel）**。

```
┌────────────────────────┐
│                        │
│                        │  宽度 = 1920 像素
│       1920×1080        │
│                        │  高度 = 1080 像素
│                        │
└────────────────────────┘

总像素数 = 1920 × 1080 = 2,073,600 ≈ 200 万像素
```

常见分辨率：

| 名称 | 分辨率 | 用途 |
|------|--------|------|
| VGA | 640×480 | 老旧设备 |
| HD / 720p | 1280×720 | 视频 |
| Full HD / 1080p | 1920×1080 | 屏幕、视频 |
| 4K UHD | 3840×2160 | 高端屏幕 |
| 相机原片 | 6000×4000 | 现代相机 |
| 全景/8K | 7680×4320 | 专业用途 |

### 1.2 短边（Short Edge）

LiteMark 以图像的**短边**作为缩放基准：

```rust
let short_edge = original_width.min(original_height) as f32;
```

**为什么用短边而非长边？**

```
横图（Landscape）：        竖图（Portrait）：
┌────────────────┐        ┌────┐
│                │        │    │
│   1920×1080    │        │    │
│                │        │1080│
└────────────────┘        │    │
    ↑ 短边=1080           │    │
                         └────┘
                         宽=1920
                         ↑ 短边=1080

两种方向的短边相同 → 边框高度一致 → 视觉体验一致
```

## 2. 比例系统设计

### 2.1 核心比例参数

LiteMark 的模板系统全部基于**比例**而非绝对像素：

```
边框高度  = 短边 × frame_height_ratio
主字号   = 边框高度 × primary_font_ratio
次字号   = 边框高度 × secondary_font_ratio
Logo 高度 = 边框高度 × logo_size_ratio
内边距   = 边框高度 × padding_ratio
```

### 2.2 比例 vs 绝对像素

**绝对像素的问题**：

```
模板设计：字体大小 = 20px

在 800×600 图像上：               在 6000×4000 图像上：
┌────────────┐                   ┌──────────────────────────┐
│            │                   │                          │
│   20px字   │  占图像 3.3%      │   20px字                 │  占图像 0.33%
│            │                   │                          │
└────────────┘                   └──────────────────────────┘
     看起来正常                         几乎看不见！
```

**比例的解决方案**：

```
在 800×600 图像上：               在 6000×4000 图像上：
┌────────────┐                   ┌──────────────────────────┐
│            │                   │                          │
│   20px字   │                   │   150px字                │
│            │                   │                          │
└────────────┘                   └──────────────────────────┘

相对比例一致，视觉感受一致
```

## 3. 分辨率缩放问题

### 3.1 硬编码上限的陷阱

LiteMark 早期版本对绝对像素值设置了上限：

```rust
let font_size = font_size.max(10.0).min(100.0);     // 字体 10~100px
let bottom_frame_height = height.max(80).min(800);  // 边框 80~800px
let padding = padding.max(5).min(50);               // 内边距 5~50px
```

这导致了一个严重问题：**高分辨率图片上的文字过小**。

### 3.2 案例分析

以 `professional` 模板处理 **6000×4600** 图片：

```
图片短边 = 4600px

边框高度设计：
  理论值 = 4600 × 0.15 = 690px
  实际值 = min(690, 800) = 690px  ✓（未触发上限）

主标题字体设计：
  理论值 = 690 × 0.26 = 179px
  实际值 = min(179, 100) = 100px   ✗（被硬上限截断！）

结果：
  文字仅占边框高度的 100/690 = 14.5%
  而设计意图是 26%
  → "大边框小文字" 的严重失调
```

### 3.3 改进方向

**方案 A：移除硬编码上限（短期）**

```rust
let font_size = font_size.max(10.0);  // 只保留下限
```

**方案 B：基准分辨率缩放（中期）**

以 1920px 短边为基准，计算缩放因子：

```rust
let resolution_scale = (short_edge / 1920.0).max(1.0);
let max_font_size = 100.0 * resolution_scale;
```

这样 1920px 以下的图保持原样，更大的图按比例放大上限。

## 4. 极端尺寸适配

### 4.1 极小图像

```rust
let bottom_frame_height = calculated_frame_height.max(80);
```

当图像短边 < 400px 时，按比例计算的边框可能只有 20~30px，文字几乎无法辨认。因此设置了 **80px 的保底高度**。

### 4.2 超大图像

```
相机原片：6000×4000
手机全景：10000×3000
扫描文档：12000×16000

对于这些超大图像，需要：
1. 确保字体上限足够大（或移除上限）
2. 考虑内存占用：6000×4000 的 RGBA 图像 ≈ 96MB
3. 输出质量与文件大小的平衡
```

## 5. 响应式设计的本质

LiteMark 的"响应式"并非网页意义上的媒体查询，而是**基于比例的数学自适配**：

```
比例链条：

短边长度
    │
    ├──→ 边框高度 = 短边 × frame_height_ratio
    │           │
    │           ├──→ 主字号 = 边框高度 × primary_font_ratio
    │           │
    │           ├──→ 次字号 = 边框高度 × secondary_font_ratio
    │           │
    │           ├──→ Logo 高度 = 边框高度 × logo_size_ratio
    │           │
    │           └──→ 内边距 = 边框高度 × padding_ratio
    │
    └──→ 一切视觉元素随图像大小自动缩放
```

这意味着：**同一份模板在 800×600 和 6000×4000 的照片上比例完全一致**。

## 6. 性能与分辨率的权衡

### 6.1 分辨率对性能的影响

LiteMark 的性能基准测试：

| 分辨率 | 耗时 | 内存占用（RGBA） |
|--------|------|-----------------|
| 800×600 | ~0.94 ms | ~1.8 MB |
| 1920×1080 | ~3.97 ms | ~7.9 MB |
| 4000×3000 | ~23.4 ms | ~45.8 MB |
| 6000×4000 | ~60 ms | ~91.6 MB |

### 6.2 优化策略

1. **避免重复解析**：字体只加载一次，复用渲染器
2. **原地修改**：直接修改图像，避免不必要的拷贝
3. **预分配内存**：创建 `Vec` 时预计算容量

```rust
let mut rgba_data = Vec::with_capacity((width * height * 4) as usize);
```

## 小结

| 概念 | 要点 |
|------|------|
| 分辨率 | 宽×高像素数，决定图像细节程度 |
| 短边基准 | 统一横图/竖图的边框高度计算 |
| 比例系统 | 所有尺寸基于比例，而非绝对像素 |
| 硬上限问题 | 高分辨率图上文字被截断，导致比例失调 |
| 保底高度 | 极小图像的 80px 最低边框高度 |
| 响应式本质 | 数学自适配，同模板任意分辨率比例一致 |
