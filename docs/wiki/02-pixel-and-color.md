# 像素、颜色模型与 Alpha 通道

> 本篇是 LiteMark 图像处理知识 Wiki 的基础篇。理解像素和颜色是读懂后续所有内容的先决条件。

---

## 1. 像素：数字图像的最小单位

### 1.1 什么是像素

**像素（Pixel）**是 "picture element" 的缩写，指数码图像中最小的可寻址单元。一张照片在屏幕上放大到极致，看到的一个个小方块就是像素。

```
┌───┬───┬───┬───┐
│ ■ │ □ │ □ │ ■ │   ← 每个小格子是一个像素
├───┼───┼───┼───┤
│ □ │ ■ │ ■ │ □ │
├───┼───┼───┼───┤
│ □ │ ■ │ ■ │ □ │      4 × 3 = 12 像素
├───┼───┼───┼───┤
│ ■ │ □ │ □ │ ■ │
└───┴───┴───┴───┘
```

一张 1920×1080 的照片，意味着它有 1920 列、1080 行，总共 **2,073,600** 个像素。

### 1.2 像素的存储方式

在计算机内存中，每个像素通常用若干个字节（Byte）来表示。常见的格式：

| 格式 | 每个像素占用 | 说明 |
|------|-------------|------|
| RGB | 3 字节 | 红、绿、蓝各 1 字节 |
| RGBA | 4 字节 | 红、绿、蓝、透明度各 1 字节 |
| 灰度 | 1 字节 | 仅亮度信息 |

1 字节 = 8 位（bit），可以表示 0~255 共 256 种强度值。

## 2. RGB 颜色模型

### 2.1 三原色叠加原理

**RGB（Red, Green, Blue）** 是一种加色模型：通过红、绿、蓝三种光的叠加产生其他颜色。

```
     红光(255,0,0)          绿光(0,255,0)          蓝光(0,0,255)
         🔴                    🟢                    🔵
          \                    |                    /
           \                   |                   /
            \      ┌───────────┴───────────┐      /
             \     │    白光(255,255,255)   │     /
              \    │         ⚪            │    /
               \   └───────────┬───────────┘   /
                \              |              /
                 \             |             /
                  \            |            /
              黄光(255,255,0)  品红(255,0,255)  青光(0,255,255)
```

### 2.2 颜色值示例

| 颜色 | RGB 值 | Hex |
|------|--------|-----|
| 纯黑 | (0, 0, 0) | `#000000` |
| 纯白 | (255, 255, 255) | `#FFFFFF` |
| 纯红 | (255, 0, 0) | `#FF0000` |
| 深灰 | (26, 26, 26) | `#1A1A1A` |
| 浅灰 | (232, 232, 232) | `#E8E8E8` |

### 2.3 Hex 颜色表示法

LiteMark 的模板系统中大量使用了 Hex 颜色：

```json
{
  "color": "#1A1A1A"
}
```

Hex 颜色是将 RGB 三个分量分别转为两位十六进制：
- `#1A1A1A` → R=0x1A=26, G=0x1A=26, B=0x1A=26
- `#FF0000` → R=0xFF=255, G=0x00=0, B=0x00=0

LiteMark 还支持 **8 位 Hex**（RGBA）：
- `#00000080` → R=0, G=0, B=0, A=0x80=128（半透明黑）

对应代码在 `renderer.rs`：

```rust
fn parse_color(color_str: &str) -> Result<Rgba<u8>, CoreError> {
    let color_str = color_str.trim_start_matches('#');
    let len = color_str.len();
    // 支持 6 位 (#RRGGBB) 和 8 位 (#RRGGBBAA)
    if len != 6 && len != 8 {
        return Err(RenderError::InvalidColor(color_str.to_string()).into());
    }
    let r = u8::from_str_radix(&color_str[0..2], 16)?;
    let g = u8::from_str_radix(&color_str[2..4], 16)?;
    let b = u8::from_str_radix(&color_str[4..6], 16)?;
    let a = if len == 8 {
        u8::from_str_radix(&color_str[6..8], 16)?
    } else {
        255  // 默认完全不透明
    };
    Ok(Rgba([r, g, b, a]))
}
```

## 3. Alpha 通道与透明度

### 3.1 什么是 Alpha

**Alpha（α）** 是 RGB 之外的第四个通道，控制像素的**透明度**：

- `Alpha = 255`（0xFF）：完全不透明
- `Alpha = 0`（0x00）：完全透明
- `Alpha = 128`（0x80）：50% 透明

```
RGBA 像素结构：
┌────────┬────────┬────────┬────────┐
│   R    │   G    │   B    │ Alpha  │
│ 1 byte │ 1 byte │ 1 byte │ 1 byte │
│  0-255 │  0-255 │  0-255 │  0-255 │
└────────┴────────┴────────┴────────┘
```

### 3.2 Alpha 混合（Alpha Blending）

当需要在一张图片上叠加另一张图片（或文字、图形）时，就要用到 **Alpha 混合**。这是 LiteMark 最核心的图像操作之一。

#### 数学公式

设：
- `Bg` = 背景像素颜色（Background）
- `Fg` = 前景像素颜色（Foreground）
- `α` = 前景的 Alpha 值，归一化为 0.0~1.0

混合后的颜色：

```
Result = Fg × α + Bg × (1 - α)
```

对 R、G、B 三个通道分别计算。

#### 直观理解

```
背景（白色）          前景（红色，50%透明）        混合结果（粉红）
┌────────┐           ┌────────┐                ┌────────┐
│ ⚪⚪⚪⚪ │   +    │ 🟥🟥🟥🟥 │    =        │ 🩷🩷🩷🩷 │
│ ⚪⚪⚪⚪ │  α=0.5 │ 🟥🟥🟥🟥 │               │ 🩷🩷🩷🩷 │
└────────┘           └────────┘                └────────┘
```

#### LiteMark 中的实现

`renderer.rs` 中的 `blend_pixel`：

```rust
fn blend_pixel(bg: Rgba<u8>, fg: Rgba<u8>) -> Rgba<u8> {
    let alpha = fg[3] as f32 / 255.0;  // 将 Alpha 归一化到 0.0~1.0
    if alpha < 0.01 {
        return bg;  // 几乎透明，直接返回背景
    }

    let r = ((fg[0] as f32 * alpha) + (bg[0] as f32 * (1.0 - alpha))) as u8;
    let g = ((fg[1] as f32 * alpha) + (bg[1] as f32 * (1.0 - alpha))) as u8;
    let b = ((fg[2] as f32 * alpha) + (bg[2] as f32 * (1.0 - alpha))) as u8;
    let a = bg[3].max(fg[3]);

    Rgba([r, g, b, a])
}
```

这个函数被用于：
1. **渐变模式**：将白色渐变层混合到原始照片上
2. **Overlay 模式**：将半透明黑色圆角矩形混合到照片上
3. **Logo 渲染**：将 Logo 的透明区域混合到目标位置

### 3.3 Alpha 在字体渲染中的应用

字体渲染也依赖 Alpha 混合。矢量字体栅格化后，每个像素产生一个 `v` 值（0.0~1.0），表示该像素被字形覆盖的程度：

```rust
outlined.draw(|px, py, v| {
    // v 是 0.0~1.0 的覆盖度，相当于 Alpha
    let r = ((color[0] as f32 * v) + (bg[0] as f32 * (1.0 - v))) as u8;
    let g = ((color[1] as f32 * v) + (bg[1] as f32 * (1.0 - v))) as u8;
    let b = ((color[2] as f32 * v) + (bg[2] as f32 * (1.0 - v))) as u8;
    image.put_pixel(px, py, Rgba([r, g, b, 255]));
});
```

这里的 `v` 就是字形的 Alpha 值，边缘的 `v` 值较小（如 0.3），形成**抗锯齿**效果。

## 4. 对比度与可读性设计

### 4.1 什么是对比度

对比度 = 前景色与背景色的亮度差异。差异越大，文字越清晰易读。

LiteMark 设计系统中的配色都经过对比度考量：

```
BottomFrame 模式（白底）：
┌─────────────────────────────┐
│                             │
│         原始照片             │
│                             │
├─────────────────────────────┤
│ 作者                    参数 │  ← 背景 = 纯白 #FFFFFF
│ 相机 · 镜头            焦距 │     文字 = 深黑 #1A1A1A
└─────────────────────────────┘     对比度极高 ✓

Overlay 模式（暗底）：
┌─────────────────────────────┐
│                             │
│         原始照片             │
│                   ┌────────┐│
│                   │ 作者   ││  ← 背景 = 半透明黑 #00000080
│                   │ 参数   ││     文字 = 纯白 #FFFFFF
│                   └────────┘│     对比度极高 ✓
└─────────────────────────────┘
```

### 4.2 LiteMark 的颜色层级

| 层级 | 用途 | 颜色 | Hex |
|------|------|------|-----|
| Level 1 | 摄影师名（最重要）| 深黑 | `#1A1A1A` |
| Level 2 | 相机·镜头 | 中深灰 | `#4A4A4A` |
| Level 3 | 光圈·快门·ISO | 中灰 | `#6A6A6A` |
| Level 4 | 分隔线 | 极淡灰 | `#E8E8E8` |

通过颜色深浅建立**视觉层级**，让读者一眼就能抓住重点。

## 5. 小结

| 概念 | 要点 | 在 LiteMark 中的体现 |
|------|------|---------------------|
| 像素 | 图像的最小单位，用字节存储颜色 | `RgbaImage` 的像素操作 |
| RGB | 红绿蓝三通道加色模型 | 所有颜色定义的基础 |
| Hex | `#RRGGBB` 或 `#RRGGBBAA` | 模板配置中的颜色字符串 |
| Alpha | 透明度通道 0~255 | 渐变、Overlay、Logo 混合 |
| Alpha 混合 | `Fg×α + Bg×(1-α)` | `blend_pixel` 函数 |
| 对比度 | 前后景亮度差 | 设计系统的四级配色 |
